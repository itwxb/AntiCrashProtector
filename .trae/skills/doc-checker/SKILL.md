# 文档更新检查

## 触发条件

当用户说以下关键词时，自动进入文档检查模式：
- "文档更新检查"
- "检查文档"
- "文档同步检查"
- "文档需要更新吗"

## 检查流程

### 1. 收集项目信息

首先读取以下文件：
- `plugin.yml` - 获取当前版本号
- `pom.xml` - 获取依赖信息
- `config.yml` - 获取配置结构

### 2. 检查文档列表

扫描项目中的所有 `.md` 文件：
```
README.md
README_EN.md
技术分析文档.md
介绍文档.md
BUILD_GUIDE.md
todo.md
```

### 3. 检查项目清单

| 检查项 | 检查内容 |
|--------|---------|
| **版本号一致性** | 所有文档中的版本号是否与 `plugin.yml` 一致 |
| **新功能文档** | 新增的 Java 类是否有对应的文档说明 |
| **配置文档** | `config.yml` 中的配置项是否都有文档说明 |
| **依赖文档** | `pom.xml` 中的新依赖是否在文档中说明 |
| **更新日志** | 是否有对应版本的更新日志 |

### 4. 输出格式

```
## 文档检查报告

### ✅ 已同步
- [文档名] 版本号正确，内容完整

### ⚠️ 需要更新
- [文档名] 缺少 xxx 内容
- [文档名] 版本号不一致 (文档: x.x.x, 实际: y.y.y)

### ❌ 缺失
- [功能名] 没有对应的文档说明
```

## 检查模板

### 版本号检查

```markdown
检查 plugin.yml 中的版本号
↓
扫描所有 .md 文件中的版本号
↓
对比是否一致
```

### 新功能检查

```markdown
扫描 src/main/java 中的 Java 文件
↓
检查文档中是否提到该类/功能
↓
列出缺失的文档说明
```

### 配置项检查

```markdown
读取 config.yml 的所有配置项
↓
检查文档中是否解释了每个配置项
↓
列出未文档化的配置项
```

## 常见问题

### 版本号位置

| 文件 | 版本号位置 |
|------|-----------|
| `plugin.yml` | `version: x.x.x` |
| `README.md` | 更新日志、配置示例 |
| `README_EN.md` | Update Log, Configuration |
| `介绍文档.md` | 更新日志、下载链接 |
| `BUILD_GUIDE.md` | 构建产物文件名 |

### 需要同步的内容

当有以下变更时，需要更新文档：

| 变更类型 | 需要更新的文档 |
|---------|---------------|
| 版本号升级 | 所有文档 |
| 新增 Java 类 | README.md, 技术分析文档.md |
| 新增配置项 | README.md, README_EN.md, 介绍文档.md |
| 新增依赖 | README.md, BUILD_GUIDE.md |
| 功能修改 | 所有相关文档 |

## 注意事项

1. **不要遗漏任何文档** - 检查所有 `.md` 文件
2. **版本号必须一致** - 所有文档中的版本号要统一
3. **中英文同步** - README.md 和 README_EN.md 内容要对应
4. **更新日志完整** - 每个版本都要有对应的更新日志
